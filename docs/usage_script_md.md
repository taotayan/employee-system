usage_script.md

สคริปต์ขั้นตอนการใช้งานระบบ Employee Management System (ตั้งแต่เริ่มต้น)

วัตถุประสงค์
- ใช้สำหรับเดโมระบบแบบ step-by-step
- ใช้เป็นคู่มือ onboarding สำหรับผู้ใช้ใหม่
- ใช้เป็น script สำหรับพรีเซนต์โปรเจกต์

--------------------------------------------------
1. เตรียมระบบก่อนเริ่ม
--------------------------------------------------
พูดกับผู้ฟัง:
"ตอนนี้เราจะเริ่มจากการเปิดระบบทั้ง Frontend และ Backend ก่อน จากนั้นจะสาธิตการสมัครสมาชิก, เข้าสู่ระบบ, จัดการพนักงาน และเมนูสำหรับแอดมินครับ"

สิ่งที่ต้องมี
- Node.js (แนะนำเวอร์ชัน 20+)
- PostgreSQL ที่มีฐานข้อมูล employee_db
- ตาราง users และ employees พร้อมใช้งาน

ไฟล์ตั้งค่าฝั่ง Backend
- ตรวจสอบไฟล์ backend/.env ให้มีค่าอย่างน้อย:
  JWT_SECRET=super_secret_key_123

--------------------------------------------------
2. ติดตั้ง dependency
--------------------------------------------------
คำสั่ง:
- Backend:
  cd backend
  npm install

- Frontend:
  cd ../frontend/ems
  npm install

พูดกับผู้ฟัง:
"หลังติดตั้งแพ็กเกจเรียบร้อย จะเริ่มเปิดเซิร์ฟเวอร์ทั้งสองฝั่งครับ"

--------------------------------------------------
3. เปิดระบบ
--------------------------------------------------
คำสั่ง:
- เปิด Backend (พอร์ต 3000):
  cd backend
  npm run dev

- เปิด Frontend (พอร์ต 5173):
  cd frontend/ems
  npm run dev

ตรวจสอบ:
- Backend: http://localhost:3000
  ควรเห็นข้อความ API is running
- Frontend: http://localhost:5173

พูดกับผู้ฟัง:
"ตอนนี้ระบบพร้อมใช้งานแล้ว โดยหน้าเว็บผู้ใช้อยู่ที่พอร์ต 5173 และ API อยู่ที่พอร์ต 3000"

--------------------------------------------------
4. เดโมการสมัครสมาชิก (Register)
--------------------------------------------------
ขั้นตอน:
1. เข้า http://localhost:5173/register
2. กรอก:
   - ชื่อผู้ใช้
   - Email
   - รหัสผ่าน
   - ยืนยันรหัสผ่าน
3. กดปุ่ม สมัครสมาชิก
4. ระบบจะพาไปหน้า Login

พูดกับผู้ฟัง:
"ผู้ใช้ใหม่สามารถสมัครสมาชิกด้วยอีเมลได้ทันที หากอีเมลซ้ำ ระบบจะแจ้งเตือน"

--------------------------------------------------
5. เดโมการเข้าสู่ระบบ (Login)
--------------------------------------------------
ขั้นตอน:
1. เข้า http://localhost:5173/login
2. กรอก Email และ Password
3. กด Login
4. ระบบพาไปหน้า Dashboard

สิ่งที่เกิดขึ้นเบื้องหลัง:
- Frontend เรียก POST /auth/login
- Backend ตรวจสอบรหัสผ่านและสร้าง JWT
- Frontend เก็บ token ใน localStorage

--------------------------------------------------
6. เดโมลืมรหัสผ่าน (Forgot Password)
--------------------------------------------------
ขั้นตอน:
1. ที่หน้า Login กดปุ่ม ลืมรหัสผ่าน?
2. จะมี Modal เด้งขึ้นมา
3. กรอก:
   - Email
   - รหัสผ่านใหม่
   - ยืนยันรหัสผ่านใหม่
4. กด บันทึกรหัสผ่านใหม่
5. เมื่อสำเร็จ ให้ login ด้วยรหัสใหม่

API ที่ใช้:
- POST /auth/reset-password

พูดกับผู้ฟัง:
"ฟีเจอร์นี้ช่วยเปลี่ยนรหัสผ่านได้จากหน้า Login โดยตรง"

--------------------------------------------------
7. เดโมหน้า Dashboard
--------------------------------------------------
ขั้นตอน:
1. หลัง Login เข้าหน้า Dashboard
2. อธิบายข้อมูลหลัก:
   - พนักงานทั้งหมด
   - จำนวนแผนก
   - วันที่เพิ่มล่าสุด (ดึงจาก created_at จริง)
3. แสดงกราฟ:
   - สัดส่วนพนักงาน
   - พนักงานแยกตามแผนก
4. เดโมปุ่ม Export:
   - Export Excel
   - Export PDF

API ที่ใช้หลัก:
- GET /employees/stats
- GET /employees/export/excel
- GET /employees/export/pdf

--------------------------------------------------
8. เดโมการจัดการพนักงาน (Employee List)
--------------------------------------------------
ขั้นตอน:
1. เข้าเมนู พนักงาน
2. ดูรายการพนักงานทั้งหมด
3. ทดลองค้นหาจากช่องค้นหา
4. ทดลอง pagination

สำหรับผู้ใช้ role = admin:
5. เพิ่มพนักงาน
6. แก้ไขพนักงาน
7. ลบพนักงาน

API ที่ใช้:
- GET /employees
- POST /employees
- PUT /employees/:id
- DELETE /employees/:id

พูดกับผู้ฟัง:
"ผู้ใช้ทั่วไปจะเห็นข้อมูลได้ แต่สิทธิ์แก้ไข/เพิ่ม/ลบ จะจำกัดเฉพาะแอดมิน"

--------------------------------------------------
9. เดโมเมนู Admin Users (เฉพาะแอดมิน)
--------------------------------------------------
เงื่อนไข:
- ต้อง login ด้วยบัญชีที่ role = admin

ขั้นตอน:
1. เข้าเมนู Admin Users
2. ดูรายชื่อผู้ใช้ทั้งหมด
3. เปลี่ยน role user/admin
4. ลบผู้ใช้ (ยกเว้นบัญชีตัวเองที่ระบบ disable ปุ่ม)

API ที่ใช้:
- GET /admin/users
- PUT /admin/users/:id/role
- DELETE /admin/users/:id

พูดกับผู้ฟัง:
"หน้านี้ถูกป้องกันสิทธิ์ด้วย role-based access control ถ้าไม่ใช่แอดมินจะเข้าไม่ได้"

--------------------------------------------------
10. เดโมการออกจากระบบ
--------------------------------------------------
ขั้นตอน:
1. กดปุ่ม ออกจากระบบ ที่แถบบน
2. ระบบลบ token และพาไปหน้า login

--------------------------------------------------
11. ปัญหาที่พบบ่อยและวิธีแก้เร็ว
--------------------------------------------------
1) หน้า API ขึ้น Cannot GET /
- ตรวจว่าเปิด URL ถูกพอร์ตหรือไม่
- หน้าเว็บให้เข้า http://localhost:5173

2) Login ขึ้น JWT_SECRET is not configured
- ตรวจ backend/.env ว่ามี JWT_SECRET
- รีสตาร์ท backend

3) พอร์ต 3000 ใช้งานอยู่ (EADDRINUSE)
- ปิด process เดิมก่อน แล้ว npm run dev ใหม่

4) เข้า Admin Users ไม่ได้
- ตรวจ role ใน token/ฐานข้อมูลว่าเป็น admin หรือไม่
- logout/login ใหม่หลังเปลี่ยน role

--------------------------------------------------
12. สรุปสำหรับพรีเซนต์ (Short Script)
--------------------------------------------------
"ระบบนี้เริ่มจากผู้ใช้สมัครสมาชิกและเข้าสู่ระบบด้วย JWT จากนั้นสามารถดู Dashboard, รายการพนักงาน, ค้นหา และ export รายงานได้ สำหรับผู้ดูแลระบบจะมีสิทธิ์เพิ่ม/แก้ไข/ลบพนักงาน และจัดการผู้ใช้งานผ่านเมนู Admin Users ได้ รวมถึงมีฟีเจอร์ลืมรหัสผ่านผ่านหน้า Login เพื่อรีเซ็ตรหัสได้ทันที"
